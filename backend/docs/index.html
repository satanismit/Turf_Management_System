API Documentation & Endpoint Reference

API Overview

Welcome to the Turf Management System API documentation. This comprehensive API allows you to manage sports turf bookings, user authentication, payments, and administrative functions.

Base URL

http://localhost:5000

Authentication

Most endpoints require JWT authentication. Include your token in the Authorization header:

Authorization: Bearer <your_jwt_token>

Quick Endpoint Reference

Authentication

  • POST /api/auth/signup
  • POST /api/auth/login
  • POST /api/auth/forgot-password
  • POST /api/auth/reset-password
  • GET /api/auth/profile
  • PUT /api/auth/profile
  • GET /api/auth/users (Admin)

Turfs

  • GET /api/turfs
  • GET /api/turfs/:id
  • POST /api/turfs (Admin)
  • PUT /api/turfs/:id (Admin)
  • DELETE /api/turfs/:id (Admin)

Bookings

  • POST /api/bookings/create
  • GET /api/bookings/user
  • PUT /api/bookings/cancel/:id
  • GET /api/bookings/all (Admin)
  • PUT /api/bookings/status/:id (Admin)

Payments

  • POST /api/payments/process
  • GET /api/payments/status/:bookingId

Comments & Reviews

  • GET /api/comments/:turfId
  • POST /api/comments
  • PUT /api/comments/:id
  • DELETE /api/comments/:id
  • GET /api/comments/:turfId/all (Admin)

Favorites

  • POST /api/auth/favorites/:turfId
  • DELETE /api/auth/favorites/:turfId
  • GET /api/auth/favorites
